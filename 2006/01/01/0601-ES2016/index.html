01、let和const

1. let

  • 说明：ES6新增let命令，用来声明变量。用法类似于var，但是let的特性更适合程序习惯；
  • let特性
    1. let变量不存在变量提升：在js代码执行之前会收集代码中var定义的变量，并赋值undefined，所以在定义变量之前使用变量，这个变量也是存在的，该现象称为变量提示；let变量不能在定义变量之前使用；
    2. 块级作用域：var变量区分函数作用域，而let变量具有块级作用域特性：即在当前的代码块中有效；
    3. 不允许重复声明相同变量：var命令可以重复声明；
    4. 不影响作用域链：如果块作用域中没有变量，会从上级块作用域寻找变量；

2. const命令

  • 说明：声明一个只读的常量。一旦声明，常量的值就不能改变。
  • 特性：同let命令
    1. 常量定义必须要赋初始值
    2. 常量值不能修改：实际上保证的并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动，引用类型中的元素是可以修改的；

3. 总结：声明变量的六种方式

  1. var
  2. let
  3. const
  4. function
  5. class
  6. import

02、解构赋值

1. 概述

  • 按照一定的模式，从数组和对象中提取值然后对变量进行赋值，这被称为解构赋值；

    在单变量的赋值表达式中，是将赋值符号右边的值赋值给左边的变量；在解构赋值的表达式中，赋值符号左边的变量是定义在和右边数据相同的数据模式中，这样右边的变量值就能赋值给对应的变量；

  • 本质上解构赋值属于“模式匹配”：只要等号两边的模式相同，右边模式中的值就会赋值给左边模式中对应的变量；

2. 数组的解构赋值

  • 模式匹配：等号两边的模式相同

  • 如果解构不成功，变量的值就等于undefined

  • 不完全结构：等号左边的模式，只匹配一部分的等号右边的数组。这种情况下，解构依然可以成功

  • 不是可遍历的解构：如果等号的右边不是可遍历的结构，那么将会报错。

  • 解构变量默认值：内部使用严格相等运算符（===），判断一个位置是否有值。所以，只有当一个数组成员严格等于undefined，默认值才会生效。如果默认值是一个表达式，那么这个表达式是惰性求值的，即只有在用到的时候，才会求值。

3. 对象解构

  • 对象解构赋值：数组的元素是按次序排列的，变量的取值由它的位置决定；而对象的属性没有次序，变量必须与属性同名，才能取到正确的值。解构失败，变量的值等于undefined。

  • 结构不相同的属性名到对应的变量名：变量依然对应对象中的属性，需要额外给变量定义别名；其实对象的解构中变量是对象属性的简写形式，属性是键值对，对象的解构赋值的内部机制，是先找到同名属性，然后再赋给对应的变量。真正被赋值的是后者，而不是前者。；

  • 嵌套对象解构：子对象所在的父属性不存在，那么将会报错；解构父属性不存在赋值为undefined，从undefined中在获取属性就会报错

  • 对象解构默认值

4. 字符串解构

  • 因为字符串可遍历，并且字符串对象也有length属性

5. 数值和布尔值解构

  • 如果等号右边是数值和布尔值，则会先转为对象。

6. 函数参数解构赋值

  • 实参赋值给形参也可以使用解构赋值，并且在参数中可以使用默认值

7. 圆括号的问题

  • 强烈建议：只要有可能，就不要在模式中放置圆括号。

  • 可以使用圆括号的情况只有一种：赋值语句的非模式部分，可以使用圆括号。

8. 结构赋值的使用场景

  • 交换变量的值
  • 从函数返回多个值
  • 函数参数的定义
  • 提取 JSON 数据
  • 函数参数的默认值
  • 遍历 Map 结构
  • 输入模块的指定方法

03、字符串扩展

1. 字符串Unicode表示法

  • 允许采用\uxxxx形式表示一个字符，其中xxxx表示字符的 Unicode 码点。只限于码点在\u0000~\uFFFF之间的字符。超出这个范围的字符，必须用两个双字节的形式表示。

  • 解读码点：只要将码点放入大括号，就能正确解读该字符。

2. 遍历字符串

  • ES6 为字符串添加了遍历器接口，可以被for...of循环遍历。

3. 模板字符串

  • 是增强版的字符串，用反引号（`）标识。它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。
